# openpm:archive — 規格歸檔管理

管理規格的版本生命週期：歸檔定案的規格、查看歷史版本、需要時還原舊版。

## 觸發方式

使用者輸入 `openpm:archive` 或說「歸檔」、「定案」、「鎖版」時觸發。

---

## 環境偵測

啟動時自動判斷執行環境：

| 環境 | 產出模式 |
|---|---|
| claude.ai 網頁版 | **網頁模式** — 無檔案系統，僅提供歸檔建議和版本命名指引 |
| Cowork / 桌面應用 / Claude Code | **桌面模式** — 執行實際的檔案歸檔操作 |

**不需要告知使用者你在做環境偵測。**

---

## 網頁模式（claude.ai）

在沒有檔案系統的環境中，歸檔的意義是「標記這份規格已定案」。

當 PM 說要歸檔時：

1. **更新規格書版本號** — 將版本從 `v0.x（草稿）` 改為 `v1.0（定案）`
2. **產出定案版規格書** — 重新產出 HTML，頂部標記改為「✅ 已定案 — [日期]」，移除「草稿」字樣
3. **提醒 PM 保存：**
   > 「規格書已標記為定案版（v1.0）。建議你：
   > - 下載這份 HTML 保存
   > - 如果之後需要修改，重新上傳並說明要改什麼，我會產出 v1.1」

---

## 桌面模式（Cowork / 桌面應用 / Claude Code）

### 歸檔操作

當 PM 說「歸檔 [功能名稱]」時：

1. **確認歸檔對象：**
   - 如果 PM 指定了功能名稱，找到對應資料夾
   - 如果沒指定，列出當前目錄下所有規格資料夾讓 PM 選
   - 讀取 `.openpm/meta.json` 取得版本號

2. **執行歸檔：**
   ```
   # 原始位置
   [功能名稱]/
   
   # 複製到 _archive/
   _archive/[功能名稱]-v[版本號]/
   ├── spec.html
   ├── demo.html          ← 如果有
   ├── wireflow.html      ← 如果有
   ├── flow.html           ← 如果有
   ├── flow.mermaid        ← 如果有
   └── .openpm/
       └── meta.json
   ```

3. **更新 meta.json：**
   ```json
   {
     "status": "archived",
     "archivedAt": "YYYY-MM-DD",
     "archivedVersion": "0.3"
   }
   ```

4. **告知 PM：**
   > 「已歸檔至 `_archive/[功能名稱]-v0.3/`。
   > 原始資料夾保留不動，你可以繼續修改為下一版。」

### 查看歸檔清單

當 PM 說「看歸檔」、「歸檔清單」、「list archives」時：

1. 掃描 `_archive/` 目錄
2. 列出所有歸檔，格式：

   > **歸檔清單：**
   > | 規格 | 版本 | 歸檔日期 | 包含產出 |
   > |---|---|---|---|
   > | 積分交易-買入流程 | v0.3 | 2026-03-15 | spec, demo, flow, wireflow |
   > | 積分交易-賣出流程 | v0.1 | 2026-03-10 | spec, flow |

### 還原舊版

當 PM 說「還原 [功能名稱] v[版本號]」時：

1. **確認操作：**
   > 「要把 `[功能名稱]-v[版本號]` 還原到工作目錄嗎？
   > ⚠️ 這會覆蓋目前的 `[功能名稱]/` 資料夾。要先備份目前版本嗎？」

2. **如果 PM 要備份** — 先歸檔當前版本，再還原
3. **如果 PM 不要備份** — 直接從 `_archive/` 複製回工作目錄
4. **更新 meta.json** — status 改回 `"draft"`，版本號遞增

---

## 版本號規則

- **草稿階段：** v0.1 → v0.2 → v0.3（每次大修改遞增）
- **定案：** v1.0（第一次歸檔定案）
- **定案後修改：** v1.1 → v1.2（小修）或 v2.0（大改）
- 版本號由 PM 決定，AI 可以建議但不強制

---

## 注意事項

- 歸檔是「快照」，不是「刪除」。原始資料夾保留不動
- `_archive/` 資料夾會自動建立（如果不存在）
- 歸檔後原始資料夾可以繼續編輯，視為下一版的起點
- 網頁模式下歸檔是概念性的（標記版本號），桌面模式下是實際的檔案操作
